Bundled-Edge Views of the Github Event Graph

All of the graphs have a node labled start. This node has an indegree2 of 0. It is usually strongly connected to repository or branch creation. The Github event graph is intersting in that almost every node is connected to every other. It is essentially a complete digraph.

The distribution of present-time events is indicated by the length of the label line, which is the log of the event's population. Its length could be interpreted in several ways. A longer line would result if the event is not followed by any subsequent events. Or it could mean a spike in the event at the moment of data capture. A stubby population line means either an unpopular event or one which is immediately followed by another action. Note the 0 population of the start event.
